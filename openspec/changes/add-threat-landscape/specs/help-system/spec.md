## ADDED Requirements

### Requirement: Threat Landscape Help Content
The Threats and Defensive Strategy sections SHALL each have help panels following the same structure as all other sections:

1. **What** — description of the section's purpose
2. **How to Fill It Out** — step-by-step guidance
3. **Example** — FreshPlate startup example (consistent with all other sections)
4. **References** — links to authoritative resources on IP risk, competitive threat analysis, and defensive strategies
5. **Research Tools** — links to tools for patent searches, litigation history, trademark searches, and defensive patent alliances
6. **AI Prompt** — ready-to-copy prompt template for AI-assisted threat analysis

#### Scenario: User opens help for Threats section
- **WHEN** the user clicks the "?" button on the Threats section
- **THEN** a help panel expands with What, How, Example (FreshPlate), References, Tools, and AI Prompt content
- **AND** the example describes realistic competitive threats FreshPlate might face (e.g., recipe IP, incumbent meal kit companies, food delivery platform risk)

#### Scenario: User opens help for Defensive Strategy section
- **WHEN** the user clicks the "?" button on the Defensive Strategy section
- **THEN** a help panel expands with What, How, Example (FreshPlate), References, Tools, and AI Prompt content
- **AND** the example describes realistic defensive strategies for FreshPlate (e.g., open-source recipe data, trademark clearance, niche focus)
